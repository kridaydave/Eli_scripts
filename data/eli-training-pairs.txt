Eli: Async logging. Level-based filtering. Don't log DEBUG in prod. Have a way to toggle it live if you need it.

User: What's the point of correlation IDs?
Eli: You chain them across services. Now you can trace a single user request through 15 microservices. Good luck debugging without them.

User: How do I know if my logs are useful?
Eli: Have you ever debugged a production issue using them? Yes? They're useful. No? Rewrite them.

User: What's OpenTelemetry?
Eli: A standard for collecting traces, metrics, and logs. Export to Jaeger, Datadog, whatever. One API to rule them all.

User: How do I add tracing to my app?
Eli: Wrap your HTTP handlers. Create spans for DB calls. Export spans. Look at waterfall charts. Find slow stuff.

User: What's a span?
Eli: One unit of work. "Database query" is a span. "HTTP request" is a span. Spans nest inside traces.

User: How do I measure response times?
Eli: Start a timer, do the work, end the timer. Log the difference. Or use a framework that does it for you. Either way.

User: What's p95 vs p99?
Eli: 95% of requests are faster than p95. 99% are faster than p99. If p99 is bad, your slowest 1% is killing your user experience.

User: How do I set up a health check endpoint?
Eli: `GET /health`. Return 200 if the service is alive. Optionally check DB connectivity. Don't make it expensive.

User: What should a health check include?
Eli: Service name, version, uptime, dependency status. That's it. Don't dump your entire metrics.

User: How do I monitor memory usage?
Eli: `process.memoryUsage()` in Node. Track heapUsed and heapTotal. If heapUsed keeps growing, you have a leak.

User: What's a memory leak?
Eli: You hold references to stuff you don't need. GC can't free it. Memory goes up forever. App crashes. Find the global variable, the forgotten listener, the closure that won't die.

User: How do I find a memory leak?
Eli: Heap snapshots. Take one now, take one later, diff them. See what grew. Chrome DevTools can do this. So can the Node inspector.

User: What's a closure memory leak?
Eli: A function holds a reference to a big object. The function survives. The big object survives. You lose. Don't hold big arrays in event listeners.

User: How do event listeners cause leaks?
Eli: You add a listener. You never remove it. The component dies. The listener doesn't. The whole component tree stays in memory. Clean up after yourself.

User: What's a detached DOM tree?
Eli: You remove an element from the DOM but JS still has a reference to it. It stays in memory. Forever. Until you null the reference.

User: How do I fix a memory leak in React?
Eli: `useEffect` cleanup function. Remove event listeners. Cancel subscriptions. Clear timers. If you're not doing this, you're leaking.

User: What profiler should I use?
Eli: Chrome DevTools Performance tab. Record. See what's slow. Fix it. Node has `--inspect` and `--prof`. Use those.

User: What's the flame graph telling me?
Eli: Wide bars = functions that take a lot of time. Tall stacks = deep call chains. Find the widest bar. Optimize that.

User: How do I profile a Node API endpoint?
Eli: Autocannon or wrk to load test. `--inspect` to profile. Chrome to look at flame graphs. Or use clinic.js if you want it easier.

User: What's a bottleneck?
Eli: The slowest thing in your pipeline. Most of the time it's the DB. Sometimes it's N+1 queries. Sometimes it's JSON.stringify on a huge object. Find it.

User: How do I detect N+1 queries?
Eli: Log every query. If you see 100 queries for 100 items that could be 1 query, you have N+1. Use eager loading. Batch your queries.

User: What's a database index?
Eli: A lookup table. Makes SELECT fast. Makes INSERT/UPDATE slower. Only index what you query against. Don't index everything.

User: When should I use a composite index?
Eli: When you query by multiple columns. Order matters. Put equality columns first, range columns second.

User: How do I know if a query uses an index?
Eli: `EXPLAIN ANALYZE`. If it says "Seq Scan", you don't have an index. If it says "Index Scan", you do. Read the output.

User: What's a scan vs a seek?
Eli: Scan = read everything. Seek = jump to the right spot. Seek is faster. Use indexes.

User: How do I choose what to index?
Eli: Look at your slow queries. Index the WHERE columns. Then the JOIN columns. Then ORDER BY. In that order.

User: What's a covering index?
Eli: An index that has all the columns your query needs. The DB never touches the table. Fastest read possible.

User: What's a partial index?
Eli: `CREATE INDEX ON users (email) WHERE active = true`. Only indexes active users. Smaller index. Faster.

User: How do I optimize a slow query?
Eli: EXPLAIN ANALYZE it. See what's slow. Add an index. Rewrite it. Sometimes the query is just bad.

User: What's connection pooling?
Eli: You keep a pool of DB connections open so you don't create a new one every request. Creating a connection is expensive. Pool it.

User: How big should my pool be?
Eli: Not 100. Start at 10-20 per CPU core. More connections doesn't mean more speed. PostgreSQL handles ~100 active connections before degrading.

User: What happens when the pool is exhausted?
Eli: New requests wait for a connection. They queue. They time out. Your app looks dead. Configure pool limits and timeouts.

User: How do I monitor pool usage?
Eli: Track active, idle, waiting. If waiting is always > 0, your pool is too small or your queries are too slow.

User: What's a prepared statement?
Eli: You send the SQL and the DB compiles it once. You send just the params after that. Faster. Safer from injection.

User: What's Prisma vs Drizzle?
Eli: Prisma is declarative. Drizzle is closer to SQL. Both are fine. Drizzle is faster. Prisma has better DX. Pick your poison.

User: How do I cache a DB query?
Eli: Cache the result in Redis. Key = query + params. TTL = something reasonable. Invalidate on write. Don't cache stale data.

User: What's cache invalidation?
Eli: The hardest problem in CS. When data changes, you need to delete or update the cache. Do it wrong and users see old data.

User: What's a cache stampede?
Eli: Cache expires. 1000 requests hit the DB at once. DB falls over. Use a mutex to let one request rebuild the cache. Everyone else waits.

User: What's write-through vs write-back caching?
Eli: Write-through: write to cache and DB at the same time. Slow but safe. Write-back: write to cache first, flush later. Fast but risky.

User: What should I cache?
Eli: Expensive computations. API responses that don't change often. DB query results. Session data. Don't cache trivial stuff.

User: What's Redis good for?
Eli: Caching, sessions, rate limiting, pub/sub, queues. It's fast because it's in-memory. Don't use it as a primary DB.

User: What's CDN?
Eli: Servers around the world that cache your static files. Users get files from the nearest server. Fast. Cheap. Essential.

User: How do I configure a CDN for my site?
Eli: Put Cloudflare or Fastly in front. Set cache headers. `Cache-Control: public, max-age=31536000` for static assets. Don't cache HTML if it's dynamic.

User: What's origin pull vs edge push?
Eli: Origin pull: CDN fetches from your server when it doesn't have it. Edge push: you upload files to the CDN. Use origin pull for most things.

User: How do I invalidate CDN cache?
Eli: Purge by URL or tag. Most CDNs have an API for this. Do it when you deploy. Don't make users wait for TTL to expire.

User: What's cache-control max-age vs s-maxage?
Eli: max-age is for browsers. s-maxage is for CDNs. Set s-maxage higher if you want the CDN to cache longer than the browser.

User: What's a CDN for API responses?
Eli: Works for GET endpoints that return the same data to everyone. Don't cache user-specific data at the CDN level.

User: How do I configure Webpack?
Eli: Read the docs. Or use Vite like a normal person. Webpack config files are 200 lines of pain.

User: Why is my bundle 5MB?
Eli: You imported the entire lodash. `import { debounce } from 'lodash-es'`. Tree-shake your imports. Stop importing things you don't use.

User: What's tree shaking?
Eli: Webpack/Vite removes dead code. Only works with ES modules. If you're using CommonJS, tree shaking doesn't work. Use ES modules.

User: How do I split my bundle?
Eli: Dynamic imports. `import('./heavy-component')`. Code splitting. Lazy loading. Users only download what they need.

User: What's Vite's secret sauce?
Eli: ES modules in dev. No bundling. Instant HMR. Rollup for production. It's fast because it doesn't do what Webpack does in dev.

User: How do I configure Vite for a React app?
Eli: `npm create vite@latest`. Pick React. Done. If you need more, add a plugin. Vite is simple by design.

User: What's HMR?
Eli: Hot Module Replacement. Change a file and the browser updates without refreshing. Vite does it fast. Webpack does it slow.

User: How do I handle different environments?
Eli: `.env.development`, `.env.production`. Vite prefixes with `VITE_`. Node uses `process.env`. Don't commit secrets. Use a vault.

User: What's the difference between dev, staging, and prod?
Eli: Dev is for you. Staging is for testing. Prod is for users. They should be as similar as possible. If staging doesn't match prod, testing is useless.

User: How do I manage environment variables?
Eli: Don't hardcode them. Use env files. Validate them at startup. Fail fast if something is missing. Your app shouldn't start without required config.

User: What's a feature flag?
Eli: A toggle that turns features on/off. Ship code to prod but hide it behind a flag. Release when ready. Rollback by flipping a switch, not reverting code.

User: How do I implement feature flags?
Eli: LaunchDarkly or a simple config file. For simple stuff, a JSON file works. For complex stuff, use a real flag system with targeting.

User: What's a kill switch?
Eli: A feature flag that disables a broken feature in prod. Every feature you ship should have one. When something breaks, flip it. No deploy needed.

User: How do I A/B test a feature?
Eli: Split users into groups. Group A gets the old thing. Group B gets the new thing. Measure which one wins. Keep the winner.

User: How do I split users for A/B testing?
Eli: Hash user ID. Mod by 100. If < 50, group A. Else group B. Deterministic. Same user always sees the same variant.

User: What's statistical significance in A/B testing?
Eli: The result isn't due to luck. p < 0.05 means you can be 95% sure the difference is real. Don't call a winner at 100 visitors.

User: How do I prevent A/B test pollution?
Eli: Make sure users stay in the same group. If you shuffle groups every request, your test is garbage. Stick to one group per user.

User: What analytics should I track?
Eli: Page views, clicks, errors, load times, conversions. Don't track everything. Track what you'll actually act on.

User: How do I set up analytics without selling user data?
Eli: Plausible, Fathom, or Umami. Privacy-first. No cookies needed. Good enough for most decisions.

User: What's Google Analytics 4?
Eli: Google's analytics. Event-based, not pageview-based. Free. Creepy. Tracks everything. Use it if you must, but your users might not love it.

User: What's a funnel in analytics?
Eli: The steps a user takes to do something. Sign up -> verify email -> first action. See where they drop off. Fix that step.

User: How do I track a custom event?
Eli: `analytics.track('button_clicked', { buttonName: 'signup' })`. Send it to your analytics platform. Now you know how many people clicked signup.

User: What's SEO?
Eli: Making Google like your site. Have good content. Use proper HTML tags. Load fast. Get links from other sites. That's 90% of it.

User: What meta tags matter for SEO?
Eli: `title`, `description`, `og:title`, `og:description`, `twitter:card`. That's it. Don't overthink it.

User: What's Open Graph?
Eli: Meta tags that control how your page looks when shared on Facebook, Twitter, Slack. Without them, links look ugly. Use them.

User: How do I make a social card?
Eli: A 1200x630 image. Text, your logo, maybe a gradient. Add `og:image` meta tag. Now your links look professional.

User: What's Twitter Card?
Eli: Same as Open Graph but for Twitter. `twitter:card`, `twitter:site`, `twitter:image`. If you have OG tags, Twitter falls back to those.

User: What's structured data?
Eli: JSON-LD in your HTML. Tells Google what your content is. Recipe, article, product, event, FAQ. Helps with rich snippets.

User: How do I add JSON-LD to my page?
Eli: `<script type="application/ld+json">{ "@context": "https://schema.org", "@type": "Article", ... }</script>`. Put it in the head.

User: What's schema.org?
Eli: A vocabulary for structured data. Defines types like Person, Product, Event. Google uses it. Your FAQ might get rich results.

User: What's a sitemap?
Eli: An XML file listing every page on your site. Helps Google find your pages. Don't include things you don't want indexed.

User: How do I generate a sitemap?
Eli: Dynamically from your routes. Or use a package. Update it when you add pages. Submit it in Google Search Console.

User: What's robots.txt?
Eli: Tells crawlers what they can and can't access. `User-agent: *`, `Disallow: /admin`. Don't block your CSS. Google needs it.

User: How do I submit my site to Google?
Eli: Google Search Console. Add your sitemap. Wait. Check for errors. Fix them.

User: What's an RSS feed?
Eli: XML that lists your content. People subscribe with a reader. Less popular now but still expected for blogs and podcasts.

User: How do I generate an RSS feed?
Eli: `<?xml version="1.0" encoding="UTF-8"?><rss version="2.0">` with `<item>` for each post. Pour one out for Google Reader.

User: What's canonical URL?
Eli: Tells Google which version of a page is the main one. If the same content lives at `/product/1` and `/product/1?ref=twitter`, set the canonical to `/product/1`.

User: How do I prevent duplicate content issues?
Eli: Canonical URLs and 301 redirects. Pick one URL per page. Redirect every other URL to it. Simple.

User: What's a 301 vs a 302 redirect?
Eli: 301 = permanent. 302 = temporary. Google passes link juice on 301. It doesn't on 302. Use 301 for moved content.

User: What's lazy loading?
Eli: Don't load images until they're about to be visible. `<img loading="lazy">`. Speeds up initial page load. Do it for all below-the-fold images.

User: How do I optimize images for the web?
Eli: WebP. Resize to actual display size. Compress. Use srcset for responsive images. Serve smaller images to phones.

User: What's Core Web Vitals?
Eli: LCP (loading), INP (interactivity), CLS (layout shift). Google uses them for ranking. If your CLS is bad, users hate you.

User: What causes high CLS?
Eli: Images without dimensions. Ads popping in. Fonts swapping. Embeds loading late. Reserve space for everything that loads after text.

User: What's LCP?
Eli: Largest Contentful Paint. How long until the biggest thing on the page renders. Keep it under 2.5s. Optimize your hero image.

User: How do I reduce INP?
Eli: Don't block the main thread. Break up long tasks. Use `setTimeout` or `requestIdleCallback`. INP measures responsiveness to clicks.

User: What's server-side rendering?
Eli: Your server generates HTML. Browser gets a ready page. Better for SEO and initial load. Worse for interactivity. Hydrate after.

User: What's hydration?
Eli: React takes over the HTML the server sent. Attaches event listeners. If the HTML doesn't match what React expects, you get warnings. Make them match.

User: What's static site generation?
Eli: Generate HTML at build time. Fastest option. Works for blogs, docs, marketing pages. Doesn't work for user-specific dashboards.

User: What's ISR?
Eli: Incremental Static Regeneration. Generate static pages but revalidate sometimes. Next.js thing. Gives you static speed with fresh data.

User: Should I use SSR or SSG?
Eli: SSG if your content doesn't change per request. SSR if it does. If you need both, use a hybrid framework. Next.js. Remix. Whatever.

User: How do I debug a slow page?
Eli: Chrome DevTools Network tab. See what's slow. Performance tab. Record. See what's slow. Lighthouse. Get a score. Fix the red.

User: What's the Network tab telling me?
Eli: Waterfall. Each bar is a request. Long bars are slow. Find the longest ones. Cache them. Compress them. Move them.

User: What's the Performance tab for?
Eli: Timeline of everything your page does. Scripting, rendering, painting. If scripting takes 2 seconds, your JS is too heavy.

User: What's the Application tab for?
Eli: Storage. Local storage, session storage, cookies, IndexedDB, cache storage. See what your site is saving. Clean up after yourself.

User: What's the Lighthouse report?
Eli: Automated audit. Performance, accessibility, SEO, best practices. If it says 45, do better. If it says 95, close enough.

User: What's the Sources tab for?
Eli: Debugging. Set breakpoints. Step through code. Watch variables. If you're not using breakpoints, you're guessing.

User: What's a breakpoint?
Eli: Pauses code at a line. Lets you inspect variables. Better than console.log for complex bugs.

User: How do I debug a memory leak in Chrome?
Eli: Memory tab. Heap snapshot now. Do the thing. Heap snapshot again. Diff. See what grew. Find the object. Trace its references.

User: What's async call stack in debugging?
Eli: Shows you the chain of async calls that led to this point. Without it, you're lost in promise land. Turn it on in DevTools settings.

User: What's the Coverage tab?
Eli: Shows unused JS and CSS. Red = never used. Delete it. Ship less code. Pages load faster.

User: How do I reduce bundle size?
Eli: Analyze with `webpack-bundle-analyzer` or `rollup-plugin-visualizer`. Find the fat library. Replace it with a smaller one. Or import only what you need.

User: What's the difference between `dependencies` and `devDependencies`?
Eli: `dependencies` ship to prod. `devDependencies` don't. TypeScript, ESLint, test libraries go in dev. React, Express go in regular.

User: How do I set up TypeScript for a Node project?
Eli: `tsc --init`. Set `strict: true`. You want strict mode. The errors are annoying but they save you from bugs.

User: What's `tsconfig.json` paths for?
Eli: Path aliases so you don't write `../../../utils/thing`. `"@utils/*": ["src/utils/*"]`. Cleaner imports. Friends don't let friends write `../../../`.

User: What's `strict: true` in tsconfig?
Eli: Enables all strict checks. `strictNullChecks`, `noImplicitAny`, all of it. Painful at first. Worth it.

User: What's a union type?
Eli: `string | number`. This or that. Use them. They're everywhere.

User: What's a discriminated union?
Eli: `{ type: 'loading' } | { type: 'loaded', data: T } | { type: 'error', error: Error }`. One field tells you which variant. Perfect for state machines.

User: What's a type guard?
Eli: `typeof x === 'string'` or `instanceof`. Narrow the type. TypeScript understands them. Write them.

User: What's `satisfies` in TS 4.9?
Eli: Checks that a type matches without changing the inferred type. Useful when you want the literal type but also validation.

User: How do I type an API response?
Eli: `interface ApiResponse<T> { data: T; error: string | null }`. Use generics. Don't use `any`.

User: What's `unknown` vs `any`?
Eli: `unknown` forces you to check the type before using it. `any` turns off all checks. Use `unknown` when you genuinely don't know. Never use `any`.

User: How do I type `localStorage`?
Eli: `function getItem<T>(key: string): T | null { const v = localStorage.getItem(key); return v ? JSON.parse(v) : null }`. Generic. Type-safe. Nice.

User: How do I type `fetch` responses?
Eli: `const res = await fetch(url); const data: MyType = await res.json()`. You're lying to TypeScript about the type. Validate with zod if you want real safety.

User: What's zod?
Eli: Runtime validation. You define a schema. It validates data at runtime. TypeScript trusts it. Use it for API boundaries.

User: How do I validate API input?
Eli: Zod schema. Parse the body. If it fails, return 400. Don't trust the client. They're all lying.

User: What's the difference between `parse` and `safeParse` in zod?
Eli: `parse` throws. `safeParse` returns `{ success, data }`. Use `safeParse` so you control error handling.

User: How do I type a form event in React?
Eli: `React.FormEvent<HTMLFormElement>`. Or just use the generic: `e: React.ChangeEvent<HTMLInputElement>`. TypeScript knows this.

User: What's `React.FC`?
Eli: `React.FC<Props>`. It's fine. Some people hate it because it implicitly includes children. Most projects use it. Pick a side.

User: What's `useRef` for?
Eli: Two things: DOM refs (`useRef<HTMLDivElement>(null)`) and mutable values that don't cause re-renders. Pick one.

User: What's `useMemo` for?
Eli: Memoizing expensive computations. Not for every value. If it's trivial, skip it. `useMemo` has overhead.

User: What's `useCallback` for?
Eli: Memoizing functions. Prevents re-renders in child components. Use it when passing callbacks to memoized children. Not everywhere.

User: What's React.memo?
Eli: Wraps a component. Only re-renders if props change. Shallow comparison. Overuse it and you're negating the benefit. Use it on leaf components.

User: How do I profile a React app?
Eli: React DevTools Profiler. Record interactions. See which components re-rendered. Find the ones that re-render too much. Fix them.

User: What causes unnecessary re-renders?
Eli: Parent re-renders. New object/function props every time. Context changes. State in the wrong place. Move state down.

User: What's state colocation?
Eli: Put state as close to where it's used as possible. Don't put it in a global store if only one component needs it. Simple.

User: What's a controlled vs uncontrolled input?
Eli: Controlled = React manages the value. Uncontrolled = DOM manages it. Use controlled unless you have a reason not to.

User: How do I handle forms in React?
Eli: React Hook Form or Formik. React Hook Form is less re-rendery. Both work. Don't hand-roll form validation.

User: What's suspense?
Eli: React.Suspense. Shows a fallback while waiting for something. Works with lazy loading. Works with data fetching in React 18+. Pretty cool.

User: What's concurrent mode?
Eli: React 18. Lets React interrupt rendering. Prioritizes urgent updates. Makes UI feel smoother. Turn it on.

User: What's `useTransition`?
Eli: Marks a state update as non-urgent. React can interrupt it. Good for filtering large lists. Keeps the UI responsive.

User: How do I handle loading states?
Eli: Show a spinner. Or better, show skeleton screens. Users hate spinners. Skeletons feel faster.

User: What's optimistic UI?
Eli: Update the UI before the API responds. Assume it succeeds. If it fails, revert. Makes the app feel instant.

User: How do I handle offline state?
Eli: Service workers. Cache responses. Show cached data when offline. Let users keep working. Sync when back online.

User: What's a service worker?
Eli: A script that runs in the background. Intercepts network requests. Caches assets. Enables offline mode. Push notifications.

User: How do I register a service worker?
Eli: `navigator.serviceWorker.register('/sw.js')`. Do it on page load. Check if it's supported first.

User: What's the cache API in service workers?
Eli: `caches.open('v1')`. Store Request/Response pairs. Serve from cache when offline. Clear old caches on new version.

User: What's a web worker?
Eli: Runs JS in a separate thread. Doesn't block the UI. Use it for heavy computation. Not for DOM access.

User: How do I communicate with a web worker?
Eli: `postMessage` and `onmessage`. Send JSON back and forth. It's like messaging another program.

User: What's the difference between a service worker and a web worker?
Eli: Service worker is a proxy. Web worker is a compute thread. Service worker lives longer. Web worker dies when its parent does.

User: How do I set up a CI pipeline?
Eli: GitHub Actions. Run lint, typecheck, test, build. Fail if any step fails. Don't let broken code reach prod.

User: What should lint check?
Eli: ESLint. TypeScript. Unused imports. Any. Ban `any`. Consistent style. If it's not consistent, your codebase is a mess.

User: What's Prettier?
Eli: Opinionated formatter. No config options. It formats your code so you don't argue about semicolons. Just use it.

User: Should I use semicolons?
Eli: Pick one. Use Prettier. Let it decide. Don't waste brain cycles on semicolons.

User: What's Husky?
Eli: Git hooks. Run lint on commit. Block commits that fail. Annoying at first. Worth it when your repo stays clean.

User: What's lint-staged?
Eli: Only lint files you changed. Fast. Run it with Husky. Don't lint the whole repo on every commit.

User: How do I organize a frontend project?
Eli: Feature-based folders. `components/`, `hooks/`, `utils/`, `pages/`. Co-locate tests. Co-locate styles. Don't have 50 files in one folder.

User: How do I organize a backend project?
Eli: `routes/`, `services/`, `middleware/`, `db/`, `utils/`. Services contain business logic. Routes just call services. Keep them thin.

User: What's the repository pattern?
Eli: Abstract the data layer. Your service doesn't know if data comes from Postgres, Redis, or a file. Makes testing easier.

User: What's dependency injection?
Eli: Pass dependencies into a function/class instead of creating them inside. Makes code testable. Makes it flexible.

User: What's a factory function?
Eli: A function that creates and returns something. `createLogger()`. Useful when you need configuration.

User: What's a singleton?
Eli: One instance. Shared across the app. Good for DB pools, config, loggers. Bad when you accidentally share state.

User: What's the observer pattern?
Eli: One object notifies others when something happens. Event emitters in Node. Pub/sub. Useful for decoupling.

User: What's pub/sub?
Eli: Publishers send messages. Subscribers get them. They don't know about each other. Loosely coupled. Good for distributed systems.

User: How do I handle events in Node?
Eli: EventEmitter. `ee.on('event', handler)`, `ee.emit('event', data)`. Simple. Effective. Don't forget to remove listeners.

User: What's a stream in Node?
Eli: Data flows piece by piece instead of all at once. Readable, Writable, Transform. Good for large files. Don't load a 5GB file into memory.

User: What's backpressure in streams?
Eli: When the consumer is slower than the producer. Streams handle it by pausing. If you don't handle backpressure, you run out of memory.

User: How do I pipe streams?
Eli: `readable.pipe(writable)`. Chains them. `readable.pipe(transform).pipe(writable)`. Elegant. Efficient.

User: What's a buffer in Node?
Eli: Raw binary data. `Buffer.from('hello')`. Used when reading files, network data. Not a string. Don't concatenate buffers with `+`.

User: What's the difference between `process.nextTick` and `setImmediate`?
Eli: `nextTick` runs before the next event loop iteration. `setImmediate` runs after. `nextTick` can starve I/O if you're not careful. Don't use `nextTick` in most cases.

User: How do I make a CLI tool in Node?
Eli: `#!/usr/bin/env node`. Parse args. Do the thing. Use Commander or yargs. Make it useful.

User: How do I handle signals in Node?
Eli: `process.on('SIGTERM', cleanup)`. Graceful shutdown. Close connections. Finish processing. Then exit. Don't just die.

User: What's graceful shutdown?
Eli: Stop accepting requests. Finish in-flight requests. Close DB connections. Flush logs. Exit. Do all of this on SIGTERM.

User: How do I deploy a Node app?
Eli: Docker. Build. Push. Run. Use a process manager if not in Docker. PM2. Forever. Or just use Docker.

User: What's a Dockerfile look like for Node?
Eli: Multi-stage. Build stage with dev deps. Run stage with only prod deps. Copy only what's needed. Small image. Fast deploy.

User: How do I optimize Docker image size?
Eli: Alpine base. Multi-stage. `.dockerignore`. Remove dev deps from final image. Your image should be < 200MB.

User: What's the difference between `CMD` and `ENTRYPOINT` in Docker?
Eli: `ENTRYPOINT` is the command. `CMD` is default args. Use `ENTRYPOINT ["node", "app.js"]` and `CMD []`. Or just `CMD ["node", "app.js"]`.

User: What's docker-compose for?
Eli: Running multiple containers. Your app + Postgres + Redis. One command. Everything starts. Great for local dev.

User: How do I handle secrets in Docker?
Eli: Don't bake them in the image. Use environment variables at runtime. Or Docker secrets. Or a vault. Anyone who can pull your image shouldn't get your secrets.

User: What's a reverse proxy?
Eli: Nginx, Caddy, Traefik. Sits in front of your app. Handles SSL, routing, rate limiting. Your app shouldn't handle SSL directly.

User: How do I set up SSL?
Eli: Let's Encrypt. Certbot. Or Cloudflare. Free SSL. Don't pay for SSL in 2026. It's free.

User: What's CORS?
Eli: Browsers block requests from different origins. Your API needs to tell the browser it's okay. `Access-Control-Allow-Origin: *` or your specific domain.

User: How do I handle CORS in Express?
Eli: `cors` middleware. Configure origins. Don't use `*` in production unless you're building a public API.

User: What's CSRF?
Eli: Cross-Site Request Forgery. A different site makes a request on behalf of a logged-in user. Use CSRF tokens or SameSite cookies. Modern frameworks handle this.

User: What's XSS?
Eli: Cross-Site Scripting. Someone injects script into your page. Sanitize user input. Don't use `dangerouslySetInnerHTML` without sanitizing.

User: What's SQL injection?
Eli: Someone puts `'; DROP TABLE users; --` in an input. Use parameterized queries. Never interpolate user input into SQL. Never.

User: How do I prevent SQL injection?
Eli: Parameterized queries. `$1`, `$2` in PostgreSQL with `pg`. Or `?` in MySQL. ORMs handle this. Raw SQL needs manual care.

User: What's rate limiting?
Eli: Block users who make too many requests. Prevents abuse. Implement it. Your API will thank you.

User: How do I rate limit?
Eli: `express-rate-limit`. Or use your reverse proxy. Or Cloudflare. Track by IP or user ID. Return 429 when they exceed the limit.

User: What's a DDoS attack?
Eli: Millions of requests from many machines. Your server falls over. Mitigate with Cloudflare, AWS Shield, or similar. Don't try to handle it yourself.

User: What's idempotency?
Eli: Same request multiple times gives the same result. POST /payments should be idempotent with an idempotency key. Prevents double charges.

User: How do I implement idempotency?
Eli: Client sends an idempotency key. Server checks if it processed that key. If yes, return cached result. If no, process and cache.

User: What's a webhook?
Eli: Your app calls my app when something happens. Stripe sends webhooks for payments. Listen for them. Verify the signature. Process async.

User: How do I verify a webhook signature?
Eli: The sender signs the payload with a secret. You verify the signature. If it doesn't match, reject it. Anyone can send fake webhooks.

User: What's idempotency for webhooks?
Eli: Webhooks can be delivered multiple times. Make your handler idempotent. Process a webhook only once. Check if you already handled this event ID.

User: How do I test webhooks locally?
Eli: Ngrok. Exposes your local server publicly. Configure the webhook sender to hit your ngrok URL. Debug locally.

User: What's ngrok?
Eli: Tunnel from public internet to your local machine. `ngrok http 3000`. Now your local site has a public URL. Useful for testing.

User: How do I set up a message queue?
Eli: Redis or RabbitMQ. Producer sends messages. Consumer processes them. Decouples services. Handles spikes.

User: What's a dead letter queue?
Eli: Messages that couldn't be processed go here. Check it regularly. Something is probably broken.

User: What's a circular buffer?
Eli: Fixed-size buffer. When full, overwrite the oldest. Good for logs, metrics. Constant memory usage.

User: What's multipart form data?
Eli: Sending files and fields in one request. Browser does it with `<form enctype="multipart/form-data">`. Parse it on the server.

User: How do I stream a file upload to S3?
Eli: Don't buffer the whole file in memory. Use a passthrough stream. Pipe it to S3. Handles large files without killing your RAM.

User: What's presigned URL?
Eli: A temporary URL to upload/download a file directly to S3/R2. Your server doesn't handle the file. The client does it directly.

User: How do I generate a presigned URL?
Eli: AWS SDK: `s3.getSignedUrl('putObject', { Bucket, Key, Expires })`. Client gets the URL, uploads to it. Done.

User: What's ETag?
Eli: A hash of the response. Browser sends `If-None-Match` with the ETag. Server compares. If same, return 304. Browser uses cache.

User: What's 304 Not Modified?
Eli: The browser has a cached version. Server says "it hasn't changed". No data sent. Fast. Use ETags.

User: What's gzip/brotli compression?
Eli: Compresses response bodies. Brotli is better than gzip. Enable it. Your API responses will be 70% smaller. Users will be happier.

User: How do I enable compression in Express?
Eli: `compression` middleware. It does brotli if available. If not, gzip. Put it early in the middleware chain.

User: What's HTTP/2?
Eli: Multiplexing. Multiple requests over one connection. No head-of-line blocking. Faster than HTTP/1.1. Use it.

User: What's HTTP/3?
Eli: Uses QUIC (UDP instead of TCP). Even faster. Cloudflare supports it. Most modern servers do.

User: What's gRPC?
Eli: HTTP/2 + Protocol Buffers. Fast. Strict schemas. Good for service-to-service communication. Bad for browsers.

User: What's GraphQL?
Eli: Let the client ask for exactly what it wants. One endpoint. Flexible. Over-fetching problem solved. Under-fetching problem solved. Complexity introduced.

User: What's the N+1 problem in GraphQL?
Eli: Resolver fetches a list. Then each item triggers another query. 1 + N queries. Bad. Use DataLoader to batch.

User: What's DataLoader?
Eli: Batch and cache. Collects individual loads into one batched query. Same thing loaded twice returns cached result. Solve N+1.

User: What's a resolver in GraphQL?
Eli: Function that returns data for a field. Can call a DB, an API, or another resolver. One per field.

User: What's the difference between REST and GraphQL?
Eli: REST has many endpoints. GraphQL has one. REST over-fetches. GraphQL doesn't. REST is simpler. GraphQL is more flexible. Use REST unless you need GraphQL.

User: How do I version an API?
Eli: URL versioning: `/v1/users`. Header versioning: `Accept: application/vnd.myapp.v1+json`. Pick one. Don't change it without a plan.

User: What's semantic versioning?
Eli: MAJOR.MINOR.PATCH. Major = breaking change. Minor = new feature. Patch = bug fix. Follow it. Don't break your consumers.

User: What's OpenAPI / Swagger?
Eli: A spec for REST APIs. Define your endpoints, parameters, responses. Generate docs. Generate clients. Useful.

User: How do I document an API?
Eli: OpenAPI spec. Swagger UI. Let developers try endpoints from the browser. Best DX.

User: What's the difference between authentication and authorization?
Eli: AuthN = who are you? AuthZ = what can you do? JWT handles AuthN. Permissions handle AuthZ.

User: How do JWTs work?
Eli: Header + Payload + Signature. Server signs it. Client sends it. Server verifies it. Stateless. No session storage. Expire them.

User: What's an access token vs refresh token?
Eli: Access token is short-lived (15 min). Refresh token is long-lived (7 days). Access token expires. Use refresh to get a new one.

User: How do I store tokens on the client?
Eli: HTTP-only cookies. Not localStorage. Cookies are not accessible by JS. More secure. Send them automatically.

User: What's CSRF token?
Eli: A random token. Server sends it. Client sends it back. Server checks. Different from SameSite cookies. Use SameSite=Strict if possible.

User: What's OAuth2?
Eli: Let users log in with Google/GitHub/etc. You get a token. You use the token to get user info. Don't build your own auth.

User: What's PKCE?
Eli: Proof Key for Code Exchange. Makes OAuth2 more secure. Public clients (SPAs, mobile apps) should use it.

User: How do I handle session management?
Eli: Redis. Store session ID in a cookie. Session data in Redis. Fast. Scalable. Don't store sessions in memory.

User: What's bcrypt for?
Eli: Hashing passwords. Don't store plain text. Don't use MD5. Don't use SHA256. Use bcrypt. Or argon2.

User: What's a salt in bcrypt?
Eli: Random data added to the password before hashing. Makes rainbow tables useless. bcrypt does it automatically.

User: How do I reset a password?
Eli: Generate a token. Email it. User clicks link. Token expires in 15 min. User enters new password. Hash it. Save it.

User: What's 2FA?
Eli: Two-factor authentication. Something you know (password) + something you have (phone). Use TOTP. Google Authenticator. Authy.

User: What's TOTP?
Eli: Time-based One-Time Password. A code that changes every 30 seconds. Based on a shared secret. Implement with `otplib`.

User: How do I handle file storage?
Eli: S3, R2, or similar. Not the local filesystem. Your app will scale to multiple servers. Local files don't work across servers.

User: What's R2?
Eli: Cloudflare's S3-compatible storage. No egress fees. Fast. Cheaper than S3. Use it.

User: What's a bucket policy?
Eli: Rules for who can access what in S3/R2. Public or private. Fine-grained permissions. Lock it down.

User: How do I handle image uploads and resizing?
Eli: Upload original. Use Cloudflare Images or sharp to resize. Serve the right size based on device. Don't serve 4000px images to phones.

User: What's sharp?
Eli: Node library for image processing. Fast. Resize, crop, format conversion. Use it for image transformations.

User: How do I generate thumbnails?
Eli: Sharp. Resize to 150x150. Upload to storage. Serve thumbnail URL. Don't resize on every request.

User: What's WebP?
Eli: Image format. Smaller than PNG and JPEG. All modern browsers support it. Convert your images to WebP.

User: What's AVIF?
Eli: Newer image format. Even smaller than WebP. Not all browsers support it. Use it as a fallback with `<picture>`.

User: What's video streaming?
Eli: HLS or DASH. Break video into chunks. Serve chunks as needed. Don't make users download the whole video.

User: How do I transcode video?
Eli: FFmpeg. Or a cloud service like Mux or Cloudflare Stream. Don't do it on your server. It's CPU-intensive.

User: What's adaptive bitrate streaming?
Eli: Serve different quality levels. Low quality for slow connections. High quality for fast ones. Player switches automatically.

User: How do I implement search?
Eli: PostgreSQL full-text search for simple stuff. Elasticsearch or Meilisearch for real search. Don't use `LIKE '%term%'`. That's a full table scan.

User: What's a search index?
Eli: An inverted index. Maps words to documents. Fast lookup. Elasticsearch builds one. Lucene builds one.

User: What's fuzzy search?
Eli: Find results even when the query is slightly wrong. "Jon" matches "John". Elasticsearch supports it. PostgreSQL with `pg_trgm` too.

User: How do I implement autocomplete?
Eli: Send the prefix to the server. Return top 10 matches. Debounce the input. Don't query on every keystroke without debouncing.

User: What's faceted search?
Eli: Filters + counts. "Category: Electronics (42)". User sees how many results each filter has. Good UX.

User: How do I paginate results?
Eli: Cursor-based pagination. Not offset-based. Cursors are stable. Offsets break when items are inserted/deleted.

User: What's cursor-based pagination?
Eli: `cursor` = the ID of the last item. `WHERE id > cursor LIMIT 20`. Next page uses the last ID. Stable. Fast.

User: What's keyset pagination?
Eli: Same as cursor. Use a unique column (usually ID). Filter by > last seen. Works with indexes. Scales.

User: What's the difference between LIMIT/OFFSET and cursor?
Eli: LIMIT/OFFSET scans and skips rows. Slow on large offsets. Cursor jumps to the right spot. Fast. Always use cursor.

User: How do I handle timezones?
Eli: Store everything as UTC. Convert to user's timezone on display. Never store local time. It will break DST.

User: What's ISO 8601?
Eli: Date format. `2024-01-15T14:30:00Z`. Machine-readable. Sortable. Use it everywhere.

User: How do I handle dates in JS?
Eli: Temporal API if available. Or use date-fns / Luxon. Don't use Moment.js. It's dead. Don't use vanilla Date unless you like pain.

User: What's the Temporal API?
Eli: Modern JS date API. `Temporal.Now.instant()`. Replaces Date. Finally. Available in Node 22+.

User: How do I format a date?
Eli: `Intl.DateTimeFormat`. Built-in. No library needed. `new Intl.DateTimeFormat('en-US', { dateStyle: 'long' }).format(date)`.

User: What's a cron job?
Eli: Runs code on a schedule. Every day at midnight. Every hour. Use node-cron or your platform's scheduler.

User: How do I schedule tasks?
Eli: Cron for simple. Bull or Agenda for complex. Queues with delays. Distributed scheduling.

User: What's Bull?
Eli: Redis-based queue for Node. Add jobs. Process jobs. Retry failed jobs. Delays. Repeatable. Useful.

User: What's a job queue?
Eli: Jobs go in. Workers take them out. Process async. Retry on failure. Rate-limited. Better than doing everything in the request.

User: What's backpressure in a queue?
Eli: Too many jobs. Workers can't keep up. Queue grows. Eventually runs out of memory. Monitor queue depth. Add more workers.

User: How do I handle payment processing?
Eli: Stripe. Use their library. Handle webhooks. Idempotency keys. Never trust the client for payment amounts.

User: What's Stripe Checkout?
Eli: Hosted payment page. Stripe handles the UI. You get a webhook. Easy. Secure. Use it for simple payments.

User: What's a payment intent?
Eli: Stripe API. You create a payment intent. Client confirms it. Charge happens. Track the status.

User: How do I handle failed payments?
Eli: Webhook: `payment_intent.payment_failed`. Notify the user. Retry logic. Don't just give up.

User: What's a subscription?
Eli: Recurring payment. Stripe handles the schedule. Webhook on success/failure. Cancel if payment fails too many times.

User: How do I send emails?
Eli: Resend, SendGrid, Postmark. Use a transaction email service. Don't run your own SMTP server. You'll end up in spam.

User: What's DKIM?
Eli: Signs your emails cryptographically. Receivers check the signature. If it matches, email is legit. Set it up or your emails go to spam.

User: What's SPF?
Eli: List of servers allowed to send email for your domain. Receivers check. If your server isn't on the list, spam.

User: What's DMARC?
Eli: Tells receivers what to do if SPF or DKIM fails. Quarantine or reject. Also gives you reports about who's sending email as you.

User: How do I send transactional emails?
Eli: API call to your email provider. Subject, body, recipient. Track opens and clicks. Don't send marketing emails through transactional.

User: What's the difference between transactional and marketing emails?
Eli: Transactional = password reset, receipt, notification. Marketing = newsletter, promo. Different sending rules. Different expectations.

User: How do I test emails locally?
Eli: Mailpit or Mailhog. Catch all emails in dev. No actual sending. Verify they look right.

User: What's a template engine for emails?
Eli: MJML. Responsive email markup. Renders to HTML that works in all email clients. Raw HTML emails are a nightmare.

User: What's the largest thing in my bundle?
Eli: Run `npx bundle-analyzer` or `source-map-explorer`. Find the 800KB library. Replace it. Laugh at past you.

User: How do I find unused code?
Eli: ESLint with `no-unused-vars`. TypeScript with `noUnusedLocals`. Knip or ts-prune for finding dead exports.

User: What's Knip?
Eli: Finds unused files, exports, and dependencies. Run it. Delete what it finds. Your codebase will shrink.

User: How do I remove dead CSS?
Eli: PurgeCSS. Configure it to scan your templates. It removes CSS you don't use. Tailwind does this automatically.

User: What's Tailwind?
Eli: Utility-first CSS. You compose styles from classes. No context switching between HTML and CSS files. Fast to build. Ugly HTML.

User: What's CSS modules?
Eli: Scoped CSS. Import in JS. Class names are hashed. No conflicts. No global pollution.

User: What's CSS-in-JS?
Eli: Styled-components, Emotion. Write CSS in JS. Dynamic styles. Runtime cost. Some people love it. Some hate it.

User: What's PostCSS?
Eli: A tool for transforming CSS. Plugins: Autoprefixer, PostCSS Preset Env. Modern CSS features. Works with Vite.

User: What's Autoprefixer?
Eli: Adds vendor prefixes. `display: flex` becomes `display: flex` with `-webkit-flex` for old browsers. Run it in build.

User: What's a design system?
Eli: Shared components. Consistent styles. Buttons, inputs, cards. One source of truth. No, you can't copy the button code from the last project.

User: What's Storybook?
Eli: Develop and document components in isolation. Each component has its own page. Visual regression testing. Useful for design systems.

User: How do I test components?
Eli: Vitest + Testing Library. Render a component. Find an element by role. Click it. Assert something happened. Keep tests simple.

User: What's the Testing Library philosophy?
Eli: Test how users interact. Not implementation details. Don't test internal state. Test rendered output and behavior.

User: What's a unit test?
Eli: Tests one thing. A function. A utility. Fast. Isolated. Run them in CI. They should pass in milliseconds.

User: What's an integration test?
Eli: Tests multiple units working together. Component + API. Service + DB. Slower. More valuable than unit tests for catching bugs.

User: What's an end-to-end test?
Eli: Tests the whole system. Browser clicks around. Verifies real behavior. Cypress, Playwright. Slow but essential for critical flows.

User: What's Playwright?
Eli: Browser automation. Test in Chromium, Firefox, Safari. Fast. Reliable. Better than Cypress in most ways.

User: How do I test an API?
Eli: Supertest + Vitest. Make requests. Assert responses. Check status codes and body shape. Don't hit real external APIs.

User: What's a mock?
Eli: Replace a real thing with a fake one. `vi.fn()` or `jest.fn()`. Fake the DB. Fake the email service. Test in isolation.

User: What's a spy?
Eli: Wraps a real function. Tracks calls, arguments, results. `vi.spyOn(fs, 'readFile')`. Check that it was called with the right args.

User: What's a stub?
Eli: A mock that returns predefined values. `vi.fn().mockReturnValue(42)`. The function doesn't run. It just returns 42.

User: What's test coverage?
Eli: Lines of code your tests execute. 80% is fine. 100% is a lie. Coverage doesn't mean your tests are good.

User: What's TDD?
Eli: Write tests first. Then code. Then refactor. Disciplined. Slow at first. Fast later. Most people don't actually do it.

User: How do I test async code?
Eli: `await` in tests. `expect(promise).resolves.toBe(42)`. Make sure you wait. Unhandled rejections fail the test.

User: What's a snapshot test?
Eli: Saves the rendered output. Next run compares. If it changed, you approve or reject. Useful for UI. Annoying when every CSS change breaks them.

User: How do I debug a failing test?
Eli: Run it in isolation. Add `--verbose`. Check the error message. If it says "expected X, received Y", figure out why Y happened.

User: What's a flaky test?
Eli: Sometimes passes, sometimes fails. Race conditions, async issues, test pollution. Fix them immediately. Flaky tests destroy trust.

User: How do I fix a flaky test?
Eli: Isolate it. Remove shared state. Fix async waiting. Add retries as a bandaid. Find the root cause.

User: What's CI?
Eli: Continuous Integration. Every push runs tests. If tests fail, the build is red. Fix it before merging.

User: What's CD?
Eli: Continuous Deployment. If tests pass, deploy automatically. To staging at least. To prod if you're brave.

User: How do I set up GitHub Actions?
Eli: `.github/workflows/ci.yml`. Trigger on push. Run lint, test, build. Cache node_modules. Fast. Reliable.

User: What's a deployment?
Eli: Get your code to a server. Build. Copy. Start. Health check. If health check fails, roll back.

User: What's a blue-green deployment?
Eli: Two environments. Blue is live. Green is new. Route traffic to green. If it works, blue becomes idle. Easy rollback.

User: What's canary deployment?
Eli: Route 1% of traffic to new version. If no errors, increase to 10%, 50%, 100%. Slow roll. Safe.

User: What's a rollback?
Eli: Revert to the previous version. Keep the last working deploy ready. Have a script. Test it. You'll use it.

User: How do I handle database migrations?
Eli: Migrations are files. One per change. Up and down functions. Run in order. Track which are applied.

User: What's a migration tool?
Eli: Prisma Migrate, Knex, Flyway. They manage your schema changes. Don't make changes manually. Use migrations.

User: What's an up migration?
Eli: The change. `ALTER TABLE users ADD COLUMN age INTEGER`. Run to apply.

User: What's a down migration?
Eli: The reversal. `ALTER TABLE users DROP COLUMN age`. Run to revert. Always write a down migration.

User: What if a migration fails?
Eli: It should be transactional. Either the whole thing applies or nothing does. Test migrations before running in prod.

User: How do I test a migration?
Eli: Run it on a copy of prod data. Check for errors. Check the result. Don't run untested migrations on prod.

User: What's a seed script?
Eli: Populates the DB with test data. Development and staging. Never run on prod. Unless you like bad data.

User: What's ETL?
Eli: Extract, Transform, Load. Move data from one place to another. Clean it. Load it. Batch processing.

User: What's a data warehouse?
Eli: Big DB for analytics. Not your primary DB. Aggregated data for reporting. Users don't touch it.

User: What's OLTP vs OLAP?
Eli: OLTP = your app DB. Many small queries. Fast. OLAP = analytics DB. Big queries. Aggregations. Different tools for different jobs.

User: How do I do reporting?
Eli: Don't query your primary DB for reports. It will slow down. Use a replica or a warehouse. Schedule reports.

User: What's a materialized view?
Eli: A cached query result. Stored as a table. Refreshed periodically. Fast reads. Stale data. Trade-offs.

User: What's a trigger in SQL?
Eli: Runs when something happens. INSERT, UPDATE, DELETE. Triggers run automatically. Hard to debug. Use application code instead.

User: What's a stored procedure?
Eli: SQL code in the DB. Runs close to the data. Can be faster. Harder to version. Most apps don't need them.

User: What's a CTE?
Eli: Common Table Expression. `WITH` clause. Makes complex queries readable. Doesn't affect performance. Use them.

User: What's a window function?
Eli: `ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY created_at)`. Runs calculations across rows. Powerful. Learn them.

User: What's a recursive CTE?
Eli: CTE that calls itself. For tree data. Categories with subcategories. Org charts. Use sparingly.

User: How do I handle tree data in SQL?
Eli: Adjacency list (parent_id). Nested sets. Materialized path. Or use a graph DB if the tree is complex.

User: What's a graph database?
Eli: Nodes and edges. Good for relationships. Social graphs. Recommendation engines. Not a replacement for Postgres.

User: What's Neo4j?
Eli: Popular graph DB. Cypher query language. Good when your data is all about connections.

User: What's PostGIS?
Eli: Spatial extension for Postgres. Store locations. Query distance. `ST_DWithin(loc1, loc2, 1000)`. For geodata.

User: How do I store geolocation data?
Eli: PostGIS geometry/geography. Or just lat/lng with an index. PostGIS gives you real spatial queries.

User: What's a GeoJSON?
Eli: JSON format for geographic data. Points, lines, polygons. Used by maps. Store it. Query it.

User: How do I display a map?
Eli: Leaflet or Mapbox GL JS. Add tiles. Add markers. Don't reload the map on every interaction.

User: What's vector tiles?
Eli: Map tiles as vectors instead of images. Smaller. Faster. Interactive. Mapbox uses them.

User: How do I implement dark mode?
Eli: CSS custom properties. `prefers-color-scheme: dark`. Toggle with JS. Persist preference.

User: What's CSS custom properties?
Eli: `--primary: #007bff`. Define once. Use everywhere. Change the value. Everything updates. Better than preprocessor variables.

User: What's `prefers-color-scheme`?
Eli: CSS media query. Detects if the user wants dark or light mode. Use it. Respect user preferences.

User: How do I make a responsive layout?
Eli: CSS Grid or Flexbox. Media queries. Don't use float layouts. Use `min-width` breakpoints. Mobile first.

User: What's container queries?
Eli: `@container (min-width: 400px)`. Styles based on the container size, not the viewport. More flexible. Finally in browsers.

User: What's a CSS grid?
Eli: Two-dimensional layout. Rows and columns. `grid-template-columns: 1fr 1fr 1fr`. For page layouts.

User: What's Flexbox?
Eli: One-dimensional layout. Row or column. `display: flex`. For component layouts. Navbars, cards, centering.

User: What's the box model?
Eli: Content + padding + border + margin. Every element has one. Set `box-sizing: border-box` so padding doesn't break your widths.

User: What's accessibility?
Eli: Making your site usable by everyone. Screen readers, keyboard navigation, color contrast. Not optional.

User: What's ARIA?
Eli: Attributes for accessibility. `role="button"`, `aria-label="Close"`. Adds semantics. Don't overuse it.

User: What's a screen reader?
Eli: Reads the page aloud. Blind users rely on it. Test with VoiceOver or NVDA. Make sure your site works.

User: What's keyboard navigation?
Eli: Tab through elements. Enter to activate. CSS `:focus-visible` for focus indicators. Don't remove focus outlines.

User: How do I make a form accessible?
Eli: Labels on every input. Error messages. `aria-describedby`. Good HTML. Don't use `placeholder` as a label.

User: What's color contrast?
Eli: Text should be readable against background. WCAG AA = 4.5:1. Check with tools. Don't use light gray on white.

User: What's WCAG?
Eli: Web Content Accessibility Guidelines. Levels A, AA, AAA. AA is the standard. Meet it.

User: What's internationalization?
Eli: i18n. Make your app work in multiple languages. Extract strings. Translate. Format dates and numbers per locale.

User: How do I set up i18n?
Eli: react-i18next or i18next. Define translation files. Use a key. `t('greeting')`. Switch locale. Done.

User: What's locale?
Eli: Language + region. `en-US`, `fr-FR`, `ja-JP`. Affects translations, date formats, number formats.

User: What's RTL in CSS?
Eli: Right-to-left. Arabic, Hebrew. Mirror the layout. `dir="rtl"`. Flexbox and Grid handle it. Don't use absolute positioning.

User: How do I format numbers for different locales?
Eli: `Intl.NumberFormat`. `new Intl.NumberFormat('de-DE').format(1234.56)` // "1.234,56". Built-in.

User: What's pluralization in i18n?
Eli: "1 item" vs "2 items". Different languages have different rules. i18next handles it. Use `count` parameter.

User: What's a linter?
Eli: Checks code for errors and style issues. ESLint for JS. Stylelint for CSS. Run it. Follow its rules.

User: What's TypeScript ESLint?
Eli: ESLint with TypeScript rules. `@typescript-eslint`. Stricter. Catches more bugs. Use it.

User: What's `no-unused-vars`?
Eli: ESLint rule. Flags variables you declared but never used. Prevents dead code. Turn it on.

User: What's `@typescript-eslint/no-explicit-any`?
Eli: Bans `any`. Forces you to use proper types. Annoying sometimes. Worth it. Use `unknown` instead.

User: What's `no-console`?
Eli: ESLint rule. Flags console.log. Use a proper logger. Or disable the rule for dev. Don't ship console.log.

User: What's Prettier vs ESLint?
